Stop Slop：给文章去除 AI 味

2026-06-04
  • AI
- 次阅读 - 位访客

你肯定见过这种 AI 写的文章：开头总得清嗓子，每段都要喊口号，动不动就"值得注意的是""不可否认的是"。

读起来像机器在模仿人，越读越烦。

Stop Slop 就是干这个的，去掉AI文章的油腔滑调。

它是 github 上开源的一个项目，目前已有 8.6k star~

# 它能干嘛？

Stop Slop 是一个规则集，告诉你 AI 写作有哪些常见毛病，怎么改。

  1. 砍废话。 "值得注意的是""不得不说""实际上"——这些词没信息量，删掉。例子："值得注意的是，这款工具确实很好用" → "这款工具好用。"

  2. 拆套路。 AI爱用一种结构：先说不是什么，再说是什么。"这不是技术问题，这是人的问题。"直接说"这是人的问题"就够了。

  3. 去掉副词。 "really""very""genuinely"都是凑数的。检查一下，90%的副词删了不影响意思。

  4. 主动语态。 "数据告诉我们"——数据不会说话。改成"我看数据发现"。主语应该是人。

  5. 别教育读者。 "你有没有想过""让我告诉你"——读者不需要你引导思考。直接把观点摆出来。

# 为什么管用

这套规则来自一个观察：AI写作的毛病是系统性的。它总在开头清嗓子，总用"不是A，是B"的对比结构，总把句子写得一样长。Stop Slop 把这些问题列成清单，写完后逐条检查就行。

并且这个项目自带评分系统：直接性、节奏感、信任度、真实感、密度，每项1-10分。低于35分就重改~

# 真实测试

我让 deepseek 生成一篇介绍广州最美地铁站的文章，要求有吸引力

然后在几个 AI 文章检测平台分别测试文章的 ai 痕迹~

# zerogpt 检测

润色后显示 human written~

# gptzero 检测

润色后 100% 人工痕迹~

# isgen.ai 检测

isgen.ai 检测润色后的文章，显示 100% 人工痕迹~

# 怎么使用

去 GitHub 搜 hardikpandya/stop-slop 就能找到。点击code -- download zip 下载到本地使用即可。

SKILL.md 是核心规则，references 文件夹里还有短语黑名单、结构套路清单和改前改后对比。

下次写完 AI 文章，再跑一遍 Stop Slop。删掉那些烦人的 AI 味，文章会好读得多。😁
